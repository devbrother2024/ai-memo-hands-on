# Story 4.3: 노트 내용 기반 자동 태그 생성

## Status

Draft

## Story

**As a** 노트 작성자,
**I want** 노트 내용을 바탕으로 AI가 자동으로 관련성 높은 태그를 생성하는 기능,
**so that** 노트를 효율적으로 분류하고 검색할 수 있으며, 관련 노트들을 쉽게 찾을 수 있다.

## Acceptance Criteria

1. 노트 내용이 50자 이상일 때 태그 생성 버튼이 표시되어야 한다
2. 태그 생성 시 3-6개의 관련성 높은 태그가 생성되어야 한다
3. 생성된 태그는 한국어와 영어 모두 지원해야 한다
4. 태그 생성 중에는 로딩 상태가 표시되어야 한다
5. 생성된 태그는 노트와 함께 저장되어야 한다
6. 태그는 개별적으로 편집/삭제가 가능해야 한다
7. 태그 재생성 기능이 제공되어야 한다
8. 중복 태그는 자동으로 제거되어야 한다
9. 태그 생성 실패 시 적절한 에러 메시지가 표시되어야 한다
10. 태그는 클릭 시 해당 태그를 가진 다른 노트로 검색이 가능해야 한다

## Dev Notes

### 아키텍처 정보

**기술 스택:**

-   Next.js 15 (App Router)
-   React 18 with TypeScript
-   Drizzle ORM with PostgreSQL (Supabase)
-   Google Gemini API (gemini-2.0-flash-001)
-   Tailwind CSS + shadcn/ui
-   Server Actions for backend logic

**AI 연동:**

-   기존 Gemini 클라이언트 재사용 (`lib/ai/gemini-client.ts`)
-   토큰 제한: 8k 토큰 (기존 요약 로직과 동일)
-   에러 핸들링: `lib/ai/errors.ts` 활용

**데이터베이스 스키마:**

```sql
-- tags 테이블
CREATE TABLE tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT UNIQUE NOT NULL,
  color TEXT, -- 태그 색상 (선택적)
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- note_tags 관계 테이블
CREATE TABLE note_tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  note_id UUID REFERENCES notes(id) ON DELETE CASCADE,
  tag_id UUID REFERENCES tags(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(note_id, tag_id)
);
```

**프롬프트 설계 가이드라인:**

-   노트 내용의 핵심 주제와 키워드 추출
-   3-6개 태그 제한 (요약과 동일한 범위)
-   한국어/영어 혼용 지원
-   일반적이고 검색에 유용한 태그 우선

**UI/UX 고려사항:**

-   요약 기능과 동일한 디자인 패턴 사용
-   태그는 색상별로 구분 가능 (선택적)
-   드래그 앤 드롭으로 태그 순서 변경 (미래 확장)
